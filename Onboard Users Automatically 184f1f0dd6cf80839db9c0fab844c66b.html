Here‚Äôs a list of key Azure Logic Apps terms:

  1. Logic App ‚Äì A cloud-based workflow automation service for integrating apps, data, and services.
  1. Workflow ‚Äì A sequence of steps that define automated processes in a Logic App.
  1. Trigger ‚Äì An event that starts a Logic App workflow (e.g., receiving an email or a timer schedule).
  1. Action ‚Äì A task or operation performed within a workflow, such as sending an email or updating a database.
  1. Connector ‚Äì A prebuilt integration component that allows Logic Apps to communicate with services like Office 365, SQL, and Azure Blob Storage.
    1. Managed Connector ‚Äì A built-in or Microsoft-maintained connector requiring authentication to interact with external services.
    1. Custom Connector ‚Äì A user-defined connector to integrate with third-party APIs not covered by built-in connectors.
  1. Built-in Actions ‚Äì Native actions available within Logic Apps without requiring external authentication.
  1. Plans:
    1. Standard Plan ‚Äì A pricing model for hosting Logic Apps within Azure App Service with enhanced networking options.
    1. Consumption Plan ‚Äì A pay-per-use pricing model where Logic Apps execute only when triggered.
    1. Integration Service Environment (ISE) ‚Äì A dedicated environment for running Logic Apps with better security, scalability, and VNET integration.
    1. Enterprise Integration Pack (EIP) ‚Äì A set of features that help integrate Logic Apps with legacy B2B protocols like AS2, X12, and EDIFACT.
  1. Stateful Workflow ‚Äì A workflow that maintains execution history and supports long-running operations.
  1. Stateless Workflow ‚Äì A lightweight workflow that executes quickly without persisting execution history.
  1. Run History ‚Äì A log of past executions, showing the status, inputs, and outputs of each Logic App run.
  1. Retry Policy ‚Äì A mechanism to automatically retry failed actions based on predefined settings.
  1. Parallel Execution ‚Äì A feature that allows multiple actions to run simultaneously within a Logic App.
  1. Controls:
    1. Loop ‚Äì A control flow construct (For Each, and Until) used to repeat actions within a Logic App.
    1. Condition ‚Äì A logical statement that determines the execution path of a workflow.
  1. Expression ‚Äì A formula using functions and variables to manipulate data dynamically within Logic Apps.
  1. Variables ‚Äì Temporary storage for values that can be modified and used within a workflow.
  1. Secure Inputs/Outputs ‚Äì A setting to mask sensitive data within Logic App runs for security.
  1. API Connection ‚Äì A resource that stores authentication details for connectors like Office 365, Salesforce, and SharePoint.
  1. Message Queues ‚Äì Services like Azure Service Bus or Storage Queues used to process messages asynchronously.
  1. Monitoring & Alerts ‚Äì Tools like Azure Monitor and Application Insights to track Logic App performance and failures.
